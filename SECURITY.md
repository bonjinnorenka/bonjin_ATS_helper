# Security Policy

## Supported Versions

セキュリティ修正は、原則として `main` ブランチ上の最新状態を対象に行います。
公開済みの過去バージョンに対するバックポートは、必要性と影響を見て個別に判断します。

## Reporting a Vulnerability

脆弱性を見つけた場合は、公開 Issue ではなく GitHub の Private Vulnerability Reporting を使って報告してください。

- Security タブから報告する
- 再現手順、影響範囲、想定される悪用方法があれば併記する
- 修正案や暫定回避策があれば添える

受領後、内容を確認して影響度と対応方針を整理します。
必要に応じて修正、リリース、公開アドバイザリの順で対応します。
